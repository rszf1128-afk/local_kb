# 本地知识库用户操作手册

## 1. 最简单用法

1. 把 `local_kb/kb_bootstrap_prompt.md` 发给 AI
2. 直接正常提问
3. 需要沉淀时直接说“总结”“复盘”“写进知识库”
4. 如果 AI 判断是新知识，先确认是否创建

## 2. 你只需要记住三种模式

- `retrieve/查用`
  - 例如：分析、排查、定位、解释
- `capture/沉淀`
  - 例如：总结、复盘、记录、写进知识库
- `package_or_governance/治理`
  - 例如：操作手册、提示词、技能、知识库结构

## 3. 推荐输入示例

```text
帮我分析一下 PX4 SITL 里船不动的问题。
把我们这次 QGC 调试总结一下，并写成知识草稿。
如果现有知识库没有这个主题，请先问我要不要创建。
```

## 4. 新知识怎么处理

如果 AI 判断当前知识库没有合适条目：

1. 不自动写正式知识
2. 先问你是否创建
3. 确认后再生成草稿或新入口

## 5. 常用命令

```bash
python3 local_kb/tools/kb_dispatch.py bootstrap-prompt
python3 local_kb/tools/kb_dispatch.py route --text "帮我总结今天的 QGC 调试"
python3 local_kb/tools/kb_dispatch.py create-from-confirmation ...
```

## 6. 一句话理解

这套系统会自动帮你做三件事：

- 查已有知识
- 沉淀稳定结论
- 治理知识库自身
把这套本地知识库整理成适合给同事用的操作方式。
```

### 示例 4：遇到新知识时

```text
这是一个新的知识主题，如果现有知识库没有，请先问我要不要创建。
```

### 示例 5：用户确认后创建草稿

```text
可以，按新笔记创建，放到 qgc 里。
```

## 6. 推荐输出格式

AI 在任何模式下，最好显式输出：

1. 当前模式
2. 当前领域
3. 读取了哪些知识
4. 复用了哪些经验
5. 下一步动作

如果脚本支持条目级输出，建议再加：

6. `best_entry/最优知识入口`
7. `related_entries/相关知识入口`
8. `suggested_next_read/下一步建议阅读`
9. `suggested_writeback/建议回写位置`

**结构化元信息**

- `layer`: `output`
- `symptom`: AI 回答里不说明当前模式、读取来源和下一步动作，导致用户难以判断它是否真的在按知识库工作
- `evidence`:
  - 显式输出模式、领域和读取来源时，更容易审查 AI 是否正确路由
  - 条目级输出能直接暴露复用的是哪条经验
- `recommended_action`:
  - 默认输出模式、领域、读取来源和下一步动作
  - 支持时增加 `best_entry`、`related_entries`、`suggested_next_read` 和 `suggested_writeback`
- `portability`: `high`
- `confidence`: `verified`

如果是沉淀模式，再加：

10. 推荐回写位置
11. 是否生成草稿

如果是新知识确认模式，再加：

12. 是否需要用户确认创建
13. 建议创建什么

如果已经进入确认后创建模式，再加：

14. 实际创建了哪些文件
15. 下一步应该补什么

## 7. 当前限制

这套系统目前是“半自动知识记忆”，不是完全自动长期记忆：

- AI 仍需要读取本地知识文件
- 草稿生成后仍建议人工审阅
- 领域知识还需要持续补充，尤其是 `QGC` 和 `embedded`
- 新领域或新知识条目默认需要用户确认后再创建

但它已经能实现比普通对话稳定得多的经验复用闭环。
