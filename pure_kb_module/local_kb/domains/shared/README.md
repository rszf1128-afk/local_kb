# 通用领域入口

这个领域用于放置跨具体技术栈的知识，例如：

## 这份文档负责什么

这份文档负责：

- 作为 `shared` 领域的总入口
- 承接跨具体技术栈的治理、方法、流程和元知识
- 告诉 AI 哪些问题应先回到通用领域而不是误落到业务域

这份文档不负责：

- 代替具体业务域承载领域细节
- 因为“暂时没分类”就无限扩成杂项堆积区
- 绕过新领域确认规则直接吞掉所有新主题

- 协作流程
- 调试方法论
- 代码审阅规范
- 提示词与知识治理经验
- 知识库系统本身的演化经验

如果某条知识暂时不适合归入明确领域，可以先作为 `shared/通用` 草稿存在。

但如果它会长期扩展成独立主题，仍应先询问用户是否创建新领域。

## 当前边界

- 当前适合放进这里的内容：
  - 协作协议
  - capture / 治理经验
  - 文档结构与回写规则
  - 跨领域复用的方法论
- 当前不适合长期放在这里的内容：
  - 明确属于 `PX4 / QGC / embedded` 的具体技术个案
  - 会持续扩展成独立主题的新领域知识

## AI 进入本领域后的最小动作

1. 先把本页作为 `shared` 领域入口
2. 再读取当前最相关的治理或规则文档
3. 如果问题已经明显收束到具体业务域，再回跳到对应领域入口
4. 如果这是一个会持续扩展的新主题，先请求用户确认是否新建领域

## 推荐回写位置

- 通用治理与方法：保留在 `shared` 正式知识
- 暂时未分类的新主题：先落 `shared` 草稿
- 已经明显长成独立主题的内容：先确认后再拆新领域
